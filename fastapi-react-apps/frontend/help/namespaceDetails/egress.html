If EgressIP is not used, the egress traffic from the pods of this namespace will use the kubernetes node ip where the pod is running and it could be any node in the cluster. If we use an external firewall to control egress traffic all the kubernetes nodes ips must be in the allow list. If the cluster is hosting multiple tenants this would would mean giving network access to sensitive backends to unrelated applications.

Egress Name ID: Identifier used to reserve and use an egress IP for this namespace. Multiple namespaces within the same application can use the same egress IP busing the same egress name id.

Enable Pod Based Egress IP: When enabled, pods may receive an egress IP based on pod-specific rules (if supported in your environment).
